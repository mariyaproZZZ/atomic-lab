# Анализ рисков подхода с неатомарными коммитами

## Основные риски:

1. **Сложность отката (revert)**
   - При неатомарном коммите откатить нужно все изменения сразу
   - Невозможно откатить только одну фичу, сохранив другую

2. **Проблемы при код-ревью**
   - Сложно понять, какое изменение к чему относится
   - Ревьюер вынужден анализировать большой объем разнородных изменений
   - Высокий риск пропустить ошибки

3. **Риск «снежного кома» зависимостей**
   - Изменения в разных частях кода переплетаются
   - Одно изменение тянет за собой другое
   - Трудно выделить причину бага

4. **Проблемы с git bisect**
   - При поиске бага бисект указывает на большой коммит
   - Трудно локализовать конкретное изменение, вызвавшее проблему
